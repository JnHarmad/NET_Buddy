# NET Buddy RAG System — Coding Standards & Best Practices

## Overview
This document defines coding standards and best practices for the NET JRF RAG study assistant built with .NET, Semantic Kernel, and Azure OpenAI.

## 2. Naming Conventions

### Classes & Interfaces
- **Service classes**: `{Entity}Service` (e.g., `EmbeddingService`, `RetrievalService`)
- **Repository/Store classes**: `{Entity}Repository` or `{Entity}Store` (e.g., `VectorStore`, `MaterialRepository`)
- **Models**: Use domain-specific names (e.g., `StudyMaterial`, `RetrievedContext`, `QueryResult`)
- **Interfaces**: Prefix with `I` (e.g., `IEmbeddingService`, `IVectorStore`)

### Constants & Configuration
- **API settings**: Use `appsettings.json` + `IOptions<T>` pattern
- **Magic numbers**: Extract to named constants (e.g., `const int ChunkSize = 800;`)
- **Enum names**: PascalCase (e.g., `enum ChunkingStrategy { Semantic, Fixed, Sentence }`)

### Variables & Methods
- **Private fields**: `_camelCase` (e.g., `_vectorStore`, `_logger`)
- **Local variables**: `camelCase`
- **Method names**: Verb + Noun (e.g., `RetrieveRelevantMaterials`, `EmbedChunks`)

## 6. Error Handling & Logging

### 6.1 Logging Pattern
```csharp
public class RetrievalService
{
    private readonly ILogger<RetrievalService> _logger;
    
    public async Task<IEnumerable<RetrievedContext>> RetrieveAsync(string query)
    {
        _logger.LogInformation("Retrieving context for query: {Query}", query);
        
        try
        {
            var embedding = await _embeddingService.EmbedAsync(query);
            var results = await _vectorStore.SearchAsync(embedding);
            _logger.LogInformation("Retrieved {Count} results", results.Count());
            return results;
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Retrieval failed for query: {Query}", query);
            throw;
        }
    }
}
```

### 6.2 Exception Handling
- Create custom exceptions for domain-specific errors:
  ```csharp
  public class MaterialIngestionException : Exception { }
  public class EmbeddingException : Exception { }
  public class RetrievalException : Exception { }
  ```
- Don't swallow exceptions; log and re-throw or handle appropriately

## 9. Performance Optimization

### 9.1 Caching Strategy
```csharp
public class CachedRetrievalService : IRetrievalService
{
    private readonly IRetrievalService _inner;
    private readonly IMemoryCache _cache;
    
    public async Task<IEnumerable<RetrievedContext>> RetrieveAsync(string query)
    {
        var cacheKey = $"retrieval:{query}";
        
        if (_cache.TryGetValue(cacheKey, out var cached))
            return (IEnumerable<RetrievedContext>)cached;
        
        var results = await _inner.RetrieveAsync(query);
        _cache.Set(cacheKey, results, TimeSpan.FromHours(1));
        return results;
    }
}
```

### 9.2 Batch Processing
- Process material ingestion in batches for large datasets
- Use parallel processing for independent operations (e.g., embedding multiple chunks)
- Monitor API rate limits

### 9.3 Lazy Loading
- Load embeddings on-demand, not upfront
- Paginate retrieval results for large result sets

## 10. NET JRF-Specific Guidelines

### 10.1 Material Organization
- Organize by subject: Physics, Chemistry, Mathematics, Life Sciences
- Tag with topic and difficulty: Basic, Intermediate, Advanced
- Track content from previous year papers separately

### 10.2 Query-Focused Prompts

- Track quiz attempts and scores

## 14. Security & Best Practices

- Validate all user inputs before using in prompts (prompt injection risk)
- Sanitize external content before embedding
- Audit API usage regularly
- Implement rate limiting for user queries
- Use HTTPS for all external communications
- Regularly rotate API keys
